A fully managed NoSQL database with single-digit millisecond performance at any scale.

📖 What is DynamoDB?

Amazon DynamoDB is a fully managed NoSQL database that stores data as key-value pairs or documents. It automatically scales to handle millions of requests per second with single-digit millisecond latency.

Unlike SQL databases, DynamoDB doesn't require you to define a rigid schema upfront. Each item can have different attributes — perfect for flexible order data.

⚙️ How Does It Work?
  • Tables — Top-level containers for your data (like "Orders")
  • Items — Individual records (like one order)
  • Partition Key — The primary key used to locate items (e.g., orderId)
  • GSI (Global Secondary Index) — Query by non-primary attributes (e.g., customerEmail)
  • On-Demand Mode — Pay per read/write, no capacity planning needed
💻 Code Example
Python — Save an order to DynamoDB
import boto3 dynamodb = boto3.resource("dynamodb", region_name="ap-south-1") table = dynamodb.Table("Orders") # Save a new order table.put_item(Item={ "orderId": "ORD-A1B2C3", "customerName": "Naman", "customerEmail": "naman@gmail.com", "item": "Paneer Burger", "status": "PLACED", "timestamp": "2026-03-29T18:00:00Z" }) # Query order history by email (using GSI) response = table.query( IndexName="EmailIndex", KeyConditionExpression=Key("customerEmail").eq("naman@gmail.com") ) orders = response["Items"]
🍔 How I Used It in QuickBite

📝 Orders Table

Every order is saved as an item with orderId (partition key), customerName, customerEmail, item, status, and timestamp.

🔍 EmailIndex GSI

A Global Secondary Index on customerEmail allows the GetOrders Lambda to instantly fetch all orders by a specific customer — powering the "My Orders" tab.
