📖 What is CloudFront?

Amazon CloudFront is a Content Delivery Network (CDN) that securely delivers your website, APIs, and streaming content to users around the world with low latency and high transfer speeds.

Instead of every user hitting your S3 bucket in Mumbai, CloudFront caches copies of your website at edge locations near each user — Tokyo, London, New York, everywhere.

⚙️ How Does It Work?
  • Origin — Your S3 bucket is the "origin" where the real files live
  • Distribution — CloudFront creates a distribution that maps to your origin
  • Edge Locations — 400+ data centers worldwide cache your content
  • HTTPS — Automatic SSL/TLS certificates for secure connections
  • Cache Invalidation — Force refresh when you update files

When a user in London visits your site, CloudFront serves it from the nearest edge — ~20ms instead of ~200ms from Mumbai!

💻 Code Example
AWS CLI — Create a Cache Invalidation
# After uploading new index.html to S3, # invalidate the CloudFront cache so users see the update: aws cloudfront create-invalidation \ --distribution-id E1234ABCDEF \ --paths "/*" # This tells CloudFront: "Forget the old cached files, # go fetch the new ones from S3."
CloudFront Distribution Settings
# Key configuration for QuickBite: Origin: quickbite-frontend.s3.amazonaws.com Protocol: HTTPS only (redirect HTTP → HTTPS) Default Root: index.html Price Class: Use All Edge Locations Cache Policy: CachingOptimized (TTL: 24 hours)
🍔 How I Used It in QuickBite

CloudFront is the gateway to QuickBite — every user accesses the app through our CloudFront URL.

🌍 Global HTTPS Delivery

CloudFront sits in front of the S3 bucket, providing a secure https://dXXXXXX.cloudfront.net URL. It serves index.html and all service pages with automatic HTTPS encryption and global caching.

⚡ Performance Boost

Users in any country get the QuickBite website from the nearest edge location instead of waiting for a round-trip to ap-south-1 (Mumbai). This reduces load times from seconds to milliseconds.
