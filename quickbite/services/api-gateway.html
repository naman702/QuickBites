📖 What is API Gateway?

Amazon API Gateway is a fully managed service that lets you create APIs that act as the "front door" for your backend services. It handles routing, throttling, authentication, and CORS — all without you managing a server.

Think of it as a smart traffic controller: when the browser sends a request to POST /order, API Gateway decides which Lambda function should handle it.

⚙️ How Does It Work?
  • Define Routes — POST /order, GET /orders, etc.
  • Attach Integrations — Each route points to a Lambda function
  • Configure CORS — Allow browsers on CloudFront to call your API
  • Deploy Stages — Publish your API with a unique URL
  • Auto-scale — Handles 1 request or 10,000 per second automatically

API Gateway gives you a URL like https://abc123.execute-api.ap-south-1.amazonaws.com that your frontend calls.

💻 Code Example
JavaScript — Frontend calling API Gateway
// POST request to place an order via API Gateway const API_URL = "https://hbalx7ocki.execute-api.ap-south-1.amazonaws.com"; const response = await fetch(`${API_URL}/order`, { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ customerName: "Naman", customerEmail: "naman@gmail.com", item: "Paneer Burger" }) }); const data = await response.json(); console.log("Order ID:", data.orderId); // → "ORD-A1B2C3"
🍔 How I Used It in QuickBite

API Gateway is the bridge between the QuickBite frontend and all backend Lambda functions.

📮 POST /order → OrderHandler Lambda

When a customer submits the order form, the browser sends a POST request to this route. API Gateway forwards the JSON body to the OrderHandler Lambda, which saves the order to DynamoDB and queues it in SQS.

📋 GET /orders → GetOrders Lambda

When a user switches to the "My Orders" tab, the browser sends a GET request with their email. API Gateway routes it to the GetOrders Lambda, which queries DynamoDB's GSI and returns their order history.

🔒 CORS Configuration

Since the frontend lives on CloudFront and the API is on a different domain, CORS headers are configured to allow cross-origin requests from the browser safely.
